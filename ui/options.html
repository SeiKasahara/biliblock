BiliBlock · B站个人屏蔽助手

默认屏蔽强度

深屏蔽在数据返回时即剔除，无闪现；浅屏蔽会折叠成「已屏蔽·点击展开」的提示条。此为默认值（也用于关键词/大模型屏蔽）；在「屏蔽名单」里可为每个人单独设深/浅。

生效范围

屏蔽方式为本地过滤，不占用B站官方拉黑名额，对方无感知。

图片评论查重（pHash · 本地）

在评论区对图片评论点 🚫（屏蔽并学习）记入「黑图库」、点「✓ 这条正常」记入白库。只存哈希、不存原图，零额度、每图几毫秒；专治"同一张恶心图被反复发"。需读取 B站图片 CDN（hdslb.com）。

图片语义相似（CLIP · 本地 WebGPU · 进阶）

本地用 WebGPU 跑 CLIP 把图编码成向量，按"与你标过的黑/白图相似度"判定，能挡"同类但换了一张"的图。首次启用从 Hugging Face 下载模型（约几十 MB，缓存后离线），需授权访问 huggingface.co。比 pHash 重——建议先用 pHash，不够再开这个。全新、谁都不像的图交给上面的「识别配图（多模态）」云端兜底。

语义相似屏蔽（文本 · 本地 WebGPU · 进阶）

本地中文文本模型（bge-small-zh）把评论与你的「🚫 屏蔽样例」编码成向量，语义距离低于阈值即自动屏蔽（同时避开与「✓ 正常」样例相近的）。首次启用从 Hugging Face 下载（约 24MB，缓存后离线），需授权 huggingface.co。阈值也能在弹窗里快速调。样例在「样例学习」里积累。

关键词屏蔽

每行一个关键词，评论/标题包含即屏蔽。零成本、零延迟，无需大模型。

正则表达式

每行一条 JavaScript 正则（不含首尾斜杠）。例如 ^.{0,2}$ 屏蔽过短评论。

大模型自动筛选评论

自动扫描评论并按你的自定义提示词判断是否屏蔽。判定结果本地缓存，重复评论不再消耗额度。

自定义提示词（筛选标准）

描述你不想看到的评论类型。模型会据此对每条评论输出 block:true/false。

开启「识别配图」后，带图/纯图评论会连同图片交给视觉模型判断（每条单独请求）。模型需支持图片输入，如智谱 glm-4v-flash（免费）、qwen-vl、gpt-4o 等。
