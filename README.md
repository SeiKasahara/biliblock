把一个 UID 加进名单，他的**投稿、评论、动态、弹幕**在所有 B 站页面对你消失。纯本地过滤，**不占用官方拉黑名额，对方无感知**。还能接入**免费大模型**按你的规则自动筛评论，并从你手动标注的样例里学习。

## 特性

- **按 UID 全站屏蔽**：评论（含楼中楼）、动态、用户投稿、搜索结果、首页/热门/相关推荐、合作投稿，弹幕，一并隐藏。
- **两级强度**：深屏蔽彻底不可见、无闪现；浅屏蔽折叠成可展开的占位条。
- **关键词 / 正则**：零成本本地过滤，不接大模型也能用。
- **大模型筛评论**：任意兼容接口，自定义提示词，批量请求 + 本地缓存 + 并发限流，默认走免费模型。
- **样例学习**：手动标注「该屏蔽 / 正常」样例，作为 few-shot 注入每次请求，还可使用WebGPU进行本地聚类分析，自定义阈值自动化naive屏蔽。
- **图片评论屏蔽（三层）**：对图片评论点 🚫，工具从你标注的图里学会挡类似的图，使用WebGPU/wasm。
- **多端同步**：名单与设置随浏览器账号同步，另支持 JSON 导入导出。
- **纯本地**：除你自己配置的大模型请求外，无任何数据外发。

## 安装

1. 下载或 `git clone` 本仓库。
2. Chrome / Edge 打开 `chrome://extensions`，开启右上角**开发者模式**。
3. 点**加载已解压的扩展程序**，选择仓库根目录。
4. 刷新任意 B 站页面。

免构建。改完代码在扩展页点该扩展的刷新按钮，再刷新页面即可，无需编译。

## 快速上手

| 操作 | 方式 |
| --- | --- |
| 屏蔽 + 学习 | 评论昵称旁点 `🚫`：拉黑该用户 + 记住这条（文本喂大模型、图片挡相似图）／ 右键用户空间链接 ／ 扩展弹窗输入 UID |

## 大模型筛选

设置页 → **大模型筛选** → 开开关 → 选服务商 → 填 API Key → **测试连接**（首次会请求接口访问权限，允许即可）。判定结果本地缓存（默认 30 天），同一条评论不重复请求。

内置服务商预设（均为 OpenAI 兼容 `/chat/completions`）：

| 服务商 | 推荐模型 | 说明 |
| --- | --- | --- |
| 智谱 GLM | `glm-4-flash` | 免费，国内直连 |
| OpenRouter | `…:free` | 免费额度，需注册 Key |
| 本地 Ollama | `qwen2.5:7b` | 离线免费、隐私最佳，需本机 `ollama serve` |
| 硅基流动 | `Qwen/Qwen2.5-7B-Instruct` | 部分模型免费 |
| Groq | `llama-3.3-70b-versatile` | 速度快，需自备网络 |
| DeepSeek / OpenAI | `deepseek-chat` / `gpt-4o-mini` | 低价，非免费 |
| 自定义 | — | 任意兼容接口 |

筛选标准由**自定义提示词**决定，可在提示词的「我的额外偏好」段追加你的个性化要求。

**带图评论**：勾选「识别配图（多模态）」后，带图/纯图评论会连同图片交给视觉模型判断（每条单独请求）。模型需支持图片输入，如 `glm-4v-flash`（免费）、`qwen-vl`、`gpt-4o`。

1. **pHash 查重**（设置页 → 常规 → 「图片评论查重」）：点 🚫 记住图的感知哈希，**同一张 / 极相似的图**被反复发时本地毫秒级挡，零模型零额度。专治"只发一张恶心图到处复读"。
2. **CLIP 图向量**（设置页 → 常规 → 「图片语义相似」）：本地 WebGPU 跑 CLIP 把图编码成向量，挡"**同类但换了一张**"的图。首次启用点「下载并预热本地模型」从 Hugging Face 拉模型（约几十 MB，缓存后离线），需授权 `huggingface.co`。运行时依赖已随仓库放在 `vendor/`（见 `vendor/README.md`）。
3. **云端视觉模型**（「识别配图（多模态）」，如 `glm-4v-flash`）：兜底**全新、谁都不像**的图。

只存哈希/向量、不存原图。需 `hdslb.com` 图片读取权限（已在 manifest 声明）。

## 降低误杀

AI 误杀多因提示词太宽。默认提示词已按「宁可漏放，不可错杀」编写，并明确列出不该屏蔽的类型（正常批评、认真讨论、玩梗、无害闲聊等）。在此之上，用**样例**进一步收敛：

```
浅屏蔽浏览  →  误杀的点「✓ 这条正常」/ 漏网的点「🚫」  →  攒够样例点「重新评估」  →  满意后切深屏蔽
```

- `🚫` 屏蔽并把评论记为「应屏蔽」样例（图片同时进黑图库）。
- 「✓ 这条正常」把评论记为「正常」样例并恢复显示。
- 样例作为你的亲自标注注入每次请求，**优先级高于模型直觉**。
- 训练期建议用**浅屏蔽**：深屏蔽会让误杀的评论彻底消失，你看不到也无从纠正。

样例在设置页 → **样例学习** 里增删，可一键「清空 AI 缓存并重新评估」。

## 工作原理

两层防线，共用同一份名单：

1. **接口拦截（主世界）**：在 `document_start` 劫持 `window.fetch`，在 B 站评论/动态/推荐等接口返回时就剔除黑名单 UID 的数据（深屏蔽无闪现）；弹幕则解析 `seg.so` 的 protobuf，用 `CRC32(uid)` 匹配 `midHash` 后丢弃。
2. **DOM 兜底（隔离世界）**：穿透 Shadow DOM 扫描评论与卡片，按 UID / 关键词 / 大模型判定隐藏或折叠，并注入 `🚫`（屏蔽并学习）按钮。

大模型分类在后台 Service Worker 完成：批量打包、并发限流、结果缓存，样例作为 few-shot 一并发送。名单与设置存于 `chrome.storage.sync`（分片以突破单项体积限制），多设备自动同步。

## 隐私

- 屏蔽逻辑全在本地浏览器内完成，名单与设置默认只存在浏览器同步存储。
- 唯一外发请求是你主动开启大模型筛选后，把评论文本发给**你自己配置**的 API。想完全离线就用本地 Ollama。
- API Key 会随账号同步到你的其它设备；介意可留空、改用本地模型。

## 目录结构

```
manifest.json
icons/
src/
  common/      crc32 · filter-core · defaults · endpoints · store
  inject/      dmproto（弹幕）· inject（fetch 拦截，主世界）
  content/     scanner（Shadow DOM 扫描）· content（编排，隔离世界）
  background/  llm（OpenAI 兼容客户端）· background（Service Worker）
ui/
  popup.*      工具栏弹窗
  options.*    设置与名单管理
```

> `filter-core-iso.js` 是 `filter-core.js` 的隔离世界专用副本，改判定逻辑时两者需同步（同一文件不能跨 world 注入两次）。
